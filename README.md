一个基于 LLM 的 AstrBot 智能回复判断插件，通过分析对话上下文和人格设定，智能判断是否应该回复消息。

## 🚀 功能特点

| 功能 | 描述 |
|------|------|
| 🔍 全消息监听 | 监听所有消息事件，支持白名单/黑名单过滤 |
| 🎭 人格设定支持 | 结合当前对话的人格角色设定进行判断 |
| 🤖 LLM 智能判断 | 基于对话上下文，调用 LLM 判断是否需要回复 |
| 📊 多维度评分 | 内容相关度、回复意愿、社交适宜性、时机恰当性四维评分 |
| ⚖️ 可配置阈值 | 通过阈值控制回复触发敏感度 |
| 🔧 模型选择 | 可选择专门的判断模型（推荐使用小模型快速判断） |
| 🔒 过滤机制 | 支持白名单/黑名单/无过滤三种模式 |
| ⚙️ 可配置 Prompt | 支持自定义判断提示词模板 |
| 🎯 优先级控制 | 可设置消息处理优先级 |

## 📦 安装方法

1. 确保已安装 AstrBot（版本 >= v4.5.7）
2. 将插件复制到 AstrBot 的插件目录（`addons/plugins/`）
   - 也可以使用 AstrBot 的插件管理器安装
   - 或下载本项目压缩包上传安装
3. 重启 AstrBot 或使用热加载命令

## ⚙️ 配置说明

插件配置分为三个部分：

### 基础设置 (basic)

| 配置项 | 类型 | 默认值 | 描述 |
|--------|------|--------|------|
| `filter_mode` | string | `none` | 消息过滤模式：`whitelist`（白名单）、`blacklist`（黑名单）、`none`（无过滤） |
| `filter_list` | list | `[]` | 会话白/黑名单列表，支持完整会话ID或群号/用户ID |
| `history_count` | int | `10` | 获取的历史消息数量上限 |
| `reply_threshold` | float | `0.6` | 回复阈值（0-1），综合评分达到此阈值才触发回复 |

### 模型设置 (model)

| 配置项 | 类型 | 默认值 | 描述 |
|--------|------|--------|------|
| `judge_provider_id` | string | `""` | 判断模型提供商，留空使用当前对话默认模型 |

> 建议选择响应快的小模型作为判断模型，以减少延迟和成本。

### 提示词设置 (prompt)

| 配置项 | 类型 | 默认值 | 描述 |
|--------|------|--------|------|
| `judge_prompt` | text | 见下方 | LLM 判断提示词模板 |

**默认判断 Prompt 模板：**

```text
你是一个智能对话判断系统。请分析以下对话上下文，判断机器人是否应该回复当前消息。

## 机器人角色设定
{persona}

## 历史对话
{history}

## 当前消息
发送者: {sender}
内容: {current_msg}

## 判断标准
请从以下维度评估（0-10分）：
1. **内容相关度**: 消息是否有趣、有价值、适合机器人回复（结合角色设定）
2. **回复意愿**: 基于角色特点，机器人是否应该主动参与此话题
3. **社交适宜性**: 在当前氛围下回复是否合适
4. **时机恰当性**: 回复时机是否恰当

回复阈值: {threshold}（综合评分达到此分数才回复）

请严格按照以下JSON格式输出结果，不要输出其他内容：
{"relevance": 分数, "willingness": 分数, "social": 分数, "timing": 分数, "should_reply": true或false, "reason": "判断理由"}
```

**支持的模板变量：**
- `{persona}` - 当前对话的人格系统提示词
- `{history}` - 对话历史
- `{current_msg}` - 当前消息内容
- `{sender}` - 发送者名称
- `{threshold}` - 回复阈值

### 过滤机制说明

- **whitelist 模式**：只处理列表中的会话
- **blacklist 模式**：屏蔽列表中的会话，处理其他所有会话
- **none 模式**：处理所有消息

列表支持以下格式：
- 完整会话ID（如 `onebot:GroupMessage:123456`）
- 群号（如 `123456`）
- 用户ID

> 使用 `/sid` 指令可获取当前会话的完整ID。

## 📝 工作原理

1. **消息监听**：通过 `@filter.event_message_type(filter.EventMessageType.ALL)` 监听所有消息
2. **过滤检查**：根据配置的过滤模式检查消息来源
3. **上下文获取**：通过 `ConversationManager` 获取对话历史
4. **人格获取**：获取当前对话的人格系统提示词
5. **构建 Prompt**：组装历史、人格、当前消息等信息
6. **LLM 判断**：调用配置的判断模型，进行多维度评分
7. **结果解析**：解析 JSON 格式的评分结果，计算综合分数
8. **回复决策**：综合评分达到阈值时触发回复

**评分维度：**
| 维度 | 说明 |
|------|------|
| 内容相关度 | 消息是否有价值、适合机器人回复 |
| 回复意愿 | 基于角色特点是否应主动参与 |
| 社交适宜性 | 当前氛围下回复是否合适 |
| 时机恰当性 | 回复时机是否恰当 |

综合评分 = (四个维度分数之和) / 40

## ⚠️ 注意事项

1. 需要在 AstrBot 中正确配置 LLM 模型才能正常使用
2. 建议选择响应快的小模型作为判断模型，以减少延迟
3. 阈值越高，回复触发越严格；阈值越低，回复越积极
4. 可通过调整优先级参数控制与其他插件的执行顺序
5. 使用 `/sid` 指令获取会话ID，方便配置过滤列表
